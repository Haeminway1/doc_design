9.2 Part 2 일정 (풀스택 시스템)

📅 Part 2 개발 타임라인

Phase 1: 학생 관리 시스템 (웹) — Month 1-3

Part 1 마이그레이션 + 다지점 + 분석
→ 3개월 차: 관리자/강사 사용 시작

Phase 2: 문제 풀이 앱 (Android) — Month 4-6

문제풀이 + 채점 + 정오표 + 오답노트
→ 6개월 차: 일부 학생 베타 테스트

Phase 3: 문제 DB + 3서비스 연동 — Month 7-9

문제 CRUD + 분류 + 3서비스 연동
→ 9개월 차: 강사 사용 시작

안정화 + 단계별 오픈 — Month 10-12

통합 테스트 + 버그 수정 + 런칭
→ 12개월 차: 전체 시스템 정식 오픈

각 Phase 완료 시 순차적으로 배포하여 중간 결과물을 확인하고 피드백을 반영합니다.

10. 자주 묻는 질문

Q1. Part 1만 하고 Part 2는 안 해도 되나요?

네, 가능합니다. Part 1만으로도 현재 스프레드시트 수기 입력 방식보다 훨씬 효율적입니다. 학생 수 500명 초과, 분점 확장, 학생용 앱, 고급 분석 필요 시 Part 2를 진행하세요.

Q2. 개발 완료 후 개발자가 그만두면요?

걱정하지 않으셔도 됩니다. 모든 소스코드가 인도되고, React/Flutter 등 업계 표준 기술 사용으로 다른 개발자가 유지보수 가능합니다. 문서화 및 저장소 접근권도 제공됩니다.

Q3. 나중에 옵션 A에서 옵션 B로 바꿀 수 있나요?

네, 가능합니다. 별도 협의를 통해 전환할 수 있으며, 기존 지급액의 일부를 크레딧으로 인정하는 방식도 협의 가능합니다.

Q4. 왜 결제 시스템은 안 만드나요?

PCI-DSS 등 금융 보안 규정 준수, PG사 계약, 결제 오류 책임 소재 문제로 전문 결제 솔루션(토스페이먼츠, 네이버페이 등) 도입을 권장합니다.

Q5. iOS 앱은 왜 안 만드나요?

개발 비용 30~50% 증가, Apple 심사 복잡도, 한국 Android 시장 점유율, 학원 태블릿 대부분 Android. iOS는 안정화 후 추후 협의 가능합니다.

Q6. Part 1 데이터를 Part 2로 옮길 수 있나요?

네, 가능합니다. 마이그레이션(이전) 작업을 통해 학생, 반, 출결, 상담 데이터를 그대로 옮길 수 있습니다.

Q7. 개발 중에 요구사항이 바뀌면요?

백로그 방식으로 유연하게 대응합니다. 매월 "이번 달 할 일"을 합의하고 우선순위를 조정하여 변화에 유연하게 대응합니다.

Q8. 옵션 A와 옵션 B, 어떤 걸 선택해야 하나요?

판단 기준 옵션 A 옵션 B
사용 목적 HLS 내부 사용 중심 외부 판매/사업화 계획
분점 배포 HLS 계열 지점만 비계열 타 학원도 포함
비용 우선 월 100만원 절약 완전한 권리 확보 우선

대부분의 학원에는 옵션 A가 적합합니다.
