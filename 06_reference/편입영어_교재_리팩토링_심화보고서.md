## 2. 기존 계획 누락/부족 항목 분석

### 2.1 ❌ 완전 누락된 항목

#### A. PDF-Unsafe 스타일 감사 및 수정 계획

기존 보고서에서 "솔리드 컬러 사용" 원칙은 언급했지만, **현재 소스 파일에서 실제로 위반하는 항목을 식별하고 수정하는 단계가 없음**.

**발견된 PDF-Unsafe 스타일:**

| 교재 | 위반 항목 | 심각도 |
|------|----------|-------|
| 문법 Bridge | `text-shadow`, `box-shadow`, `rgba()`, `backdrop-filter: blur` | 높음 |
| 문법 Advanced | `box-shadow`, `rgba()` | 중간 |
| 독해 Basic | `text-shadow`, `rgba(46,24,71,0.5)` 오버레이 | 높음 |
| 독해 Bridge | `rgba()` 투명도, `filter: saturate(0.8)` | 중간 |
| 구문독해 | `text-shadow`, `box-shadow`, `rgba()` | 높음 |
| 보카 | `text-shadow`, `rgba(255,255,255,0.85)` | 높음 |
| 논리 | `linear-gradient(135deg,...)`, `hover` 효과, `rgba()` | **치명적** |

**→ 추가 필요: Phase 1에 "PDF-Unsafe 스타일 → Safe 대체 매핑표" 작성 단계**

```
필요한 매핑:
- linear-gradient(...)  → 솔리드 background-color
- rgba(r,g,b,a)        → 불투명 hex (#RRGGBB)
- text-shadow           → font-weight + color 조합
- box-shadow            → border
- backdrop-filter       → background-color
- filter: saturate()    → 원본 색상 조정
- :hover 효과           → 제거 (PDF에서 무의미)
```

#### B. 폰트 로딩 방식 통일 계획

현재 교재별로 폰트 로딩이 제각각:

| 방식 | 교재 |
|------|------|
| `@import url(...)` | 문법 Bridge |
| `<link href="...">` | 독해 전체, 보카, 논리 |
| 로딩 없음 (시스템 폰트 의존) ⚠️ | 구문독해 |

**→ 추가 필요: 통일 폰트 로딩 전략 (`<link>` + preconnect 표준)**

#### C. 페이지 정렬 일관성 검사

**독해 Bridge가 좌측 정렬** (`margin: auto` 없음) — 유일하게 A4 중앙에 오지 않는 교재.

**→ 추가 필요: 모든 교재의 `.page { margin: Xmm auto }` 강제 규칙**

#### D. 색상 버그 수정

**2-5.html에서 `--color-sandy-beige: #F2C944`** — 올바른 값은 `#F2C994`.

**→ 추가 필요: Phase 2 전에 기존 소스의 알려진 버그 수정 단계**

#### E. 논리 교재 전용 컴포넌트

기존 보고서는 논리 교재의 고유 컴포넌트를 식별하지 않음:

- `.step-section` / `.step-header` / `.step-content` — 단계별 학습
- `.pattern-box` / `.connectors` / `.connector` — 논리 연결어 패턴
- `.note-box` (emoji ::before) — 참고 박스
- `.contents-grid` / `.content-item` — 그리드형 목차

**→ 추가 필요: 논리 교재 전용 컴포넌트를 `components/` 목록에 반영**

#### F. 보카 교재 전용 컴포넌트

기존 보고서에 보카 컴포넌트는 `.word-entry`만 언급. 누락:

- `.day-title-page` / `.day-number` / `.day-theme` — Day별 시작 페이지
- `.veras-tip-page` — Vera's 학습 팁 전용 페이지
- `#system-info` (개발 디버그 패널 — **제거 대상**)
- 이중 카운터 시스템 (`page-counter` + `word-counter`) — 정리 필요

#### G. 구문독해 전용 컴포넌트

- `.part-opener` / `.part-title` / `.part-subtitle` — Part 시작 페이지
- `.unit-header` / `.testing-point-header` / `.meta-info` — Unit 구조
- `.concept-box` vs `.rule-box` vs `.trap-box` — 3종 박스 (통일 vs 유지 결정 필요)
- `.example-analysis` — 구문 분석 예시 (기존 `example-block`과 다른 구조)

### 2.2 ⚠️ 부족한 항목 (언급은 있으나 깊이 부족)

#### A. 템플릿 매핑이 5개 → 실제 필요한 것은 6개

기존 보고서에서 5개 템플릿으로 수렴 가능하다고 했으나, 실제 분석 결과:

| 템플릿 ID | 교재 | 메인 컬러 | 기존 보고서 | 실제 상태 |
|----------|------|----------|-----------|----------|
| `ocean-blue` | 문법 Bridge + Advanced | `#345C85` | ✅ 언급 | ⚠️ Advanced는 현재 완전히 다른 디자인 |
| `royal-purple` | 독해 Basic | `#9B59B6` | ✅ 언급 | ✅ |
| `earth-tone` | 독해 Bridge | `#4B5358` | ✅ 언급 | ⚠️ Pantone 2025 색상이라 특수함 |
| `sky-academic` | 독해 Intermediate + 보카 + 논리 | `#3498DB` | ✅ 언급 | ⚠️ 3개 교재 묶기엔 디자인 차이 큼 |
| `mint-sky` | 구문독해 | `#63b3ed` | ✅ 언급 | ✅ |

**문제점:**
- `sky-academic`으로 묶인 3개 교재(독해 Intermediate, 보카, 논리)의 **실제 디자인 차이가 큼**:
  - 독해 Intermediate: 깔끔하고 미니멀, 패딩 작음 (12mm)
  - 보카: Tailwind 기반 Slate 계열, 대형 타이틀 (96px)
  - 논리: gradient 사용, step 구조, 가장 불완전

**→ 개선안:** `sky-academic`을 `sky-clean`(독해 Inter+보카)과 `logic-blue`(논리 전용)로 분리 검토. 또는 보카 교재만 Day 구조가 너무 특수하므로 별도 `vocab-sky` 템플릿 고려.


#### D. 빌드 스크립트 설계 깊이 부족

기존 보고서에서 `build/assemble.js`의 역할은 언급했으나 실제 구현 고려사항이 빠짐:

1. **배경 이미지 경로 해결**: 소스에서 상대 경로 → 빌드 시 절대 경로로 변환해야 함
2. **폰트 로딩 주입**: 각 교재 HTML에 일관된 `<link>` 태그 자동 삽입
3. **CSS 파일 주입 순서**: `vera-core.css` → `components/` → `templates/{id}/` 순서 보장
4. **카운터 초기화**: 페이지 번호 카운터를 교재별로 초기화하는 로직
5. **Puppeteer PDF 옵션**: 교재별로 다른 헤더/푸터 설정이 필요할 수 있음
6. **오버플로 검증**: `validate-overflow.js`의 구체적 구현 방안 미정의

---

### 2.3 ✅ 기존 계획에서 잘 된 부분

- 하이브리드 아키텍처 (JSON + HTML) 결정: 적절함
- JSON Schema 설계: 충분히 상세함
- 매니페스트(YAML) 기반 조립: 좋은 접근
- 템플릿 시스템의 레이어 구조 (base → components → templates): 견고함
- 네이밍 컨벤션 정의: 일관성 있음
