# 피드백지 제작 가이드라인 v1.0

> 학습 피드백지의 일관된 디자인과 구조를 보장하기 위한 규칙 문서.
> 템플릿 참조: `03_system/templates/피드백지_template.html`

---

## 1. 구조 규칙 (필수)

### 절대 금지
- **표지 페이지 금지**: 어떤 경우에도 표지를 만들지 않는다
- **목차 금지**: 목차 섹션을 포함하지 않는다
- **"전문 추출" 표현 금지**: "문제 리스트" 또는 "틀린 문제 리스트"로 대체

### 필수 구조 (순서대로)

```
1. 첫 페이지: 리포트 헤더 + 학습 요약
2. 틀린 문제 리스트 (section-header bar)
3. 해설 파트 (section-header bar, 테마별)
4. 연습문제 (section-header bar, 테마별 20문제씩, 문제만)
5. 정답 및 해설 (별도 페이지, 테마별)
```

---

## 2. 첫 페이지 구성

첫 페이지는 표지 대신 **학습 진단 리포트** 형식으로 시작한다.

### 리포트 헤더 (.report-header)
```html
<div class="report-header">
  <div class="report-date">2026. 02. 14</div>
  <h1 class="report-title">학습 피드백지</h1>
  <div class="report-subtitle">문법 — UNIT 05 형용사/부사</div>
  <div class="report-meta">
    <span><strong>학생</strong> 조근영</span>
    <span><strong>교재</strong> Vera's Flavor 편입문법</span>
    <span><strong>범위</strong> UNIT 05 심화 연습</span>
  </div>
</div>
```

### 학습 요약 카드 (.diagnosis-card)
반드시 포함해야 할 내용:
- **전체 정답률** 및 틀린 문제 수
- **잘한 점**: 구체적으로 칭찬 (예: "2형식 동사 보어 문제는 대부분 정확히 판별")
- **아쉬운 점**: 개선이 필요한 영역 (예: "hard/hardly 혼동쌍에서 실수 반복")
- **오답 패턴 분석 테이블**: 테마별 빈도 + 심각도 배지
- **학습 우선순위**: priority-order 박스로 표시

### 심각도 배지
```html
<span class="severity-high">높음</span>
<span class="severity-mid">보통</span>
<span class="severity-low">낮음</span>
```

## 3. 섹션별 상세 규칙

### 섹션 1: 틀린 문제 리스트
- `section-header` bar로 시작
- 각 문제를 `problem-card` 안에 배치
- `question-box`로 원문, `answer-box`로 정답 표시
- 제목은 "틀린 문제 리스트" (NOT "전문 추출", NOT "발견된 항목 총정리")

### 섹션 2: 해설 파트

**과목에 따라 해설 형식이 완전히 다르다. 반드시 구분할 것.**

#### 2-A. 문법 / 구문독해 해설
- 테마별로 구분 (h2 단위)
- `concept-box`: 핵심 개념
- `grammar-block`: 문법 구조/예문 (pre-wrap)
- `tip-box`: 학습 팁, 판별법
- `warning-box`: 주의사항, 함정
- 비교표, 판별 순서도(flowchart) 적극 활용

#### 2-B. 독해 해설 (지문별 개별 분석)

**절대 규칙**: 독해 문제는 "추론 전략", "독해 풀이법" 같은 추상적 문법 설명으로 해설하지 않는다.
틀린 지문을 **하나씩 개별 분석**하여 지문 자체를 이해시키는 것이 목적이다.
**본문은 양쪽 정렬(text-align: justify)을 적용한다.**

각 틀린 지문에 대해 아래 **5단계 구조**를 따른다:

**① 한줄 요약**
지문의 핵심 주제를 1-2문장으로 압축 요약한다.
```html
<div class="concept-box">
  <h4>1. 한줄 요약</h4>
  <p>꿀벌은 대규모 군집 안에서 여왕벌, 일벌, 수벌이 각각의 역할을 수행하며,
  특히 꿀벌의 감소가 생태계에 미치는 영향이 심각하다는 내용이다.</p>
</div>
```

**② 전체 해석 (Full Translation)**
영어 지문 전체를 자연스러운 한국어로 번역한다. 직역보다 의역 중심.
```html
<div class="grammar-block">
  <h4>2. 전체 해석 (Full Translation)</h4>
  <p>꿀벌은 매혹적인 곤충이다. 그들은 군집(colony)이라 불리는 큰 집단 안에서 함께 살며,
  군집 안의 모든 벌은 특정한 역할을 가지고 있다. 여왕벌은 군집의 어머니이다. ...</p>
</div>
```

**③ 핵심 키워드**
지문의 핵심 어휘/표현을 테이블로 정리한다. "정답 방향"은 정답과 일치하는 표현, "반대 방향"은 반대되는 표현.
```html
<table>
  <thead>
    <tr><th>구분</th><th>표현 및 키워드</th></tr>
  </thead>
  <tbody>
    <tr><td>핵심 키워드</td><td>colony, specific job, queen bee, worker bees, drones</td></tr>
    <tr><td>정답 방향 (Positive)</td><td>live together, specific job, defend the hive, produce honey</td></tr>
    <tr><td>반대 방향 (Negative)</td><td>disappearing, serious problem, ecosystem</td></tr>
  </tbody>
</table>
```

**④ 구조 및 방향성**
글의 전개 방향(POSITIVE/NEGATIVE)과 구조 유형을 분석한다.
```html
<div class="tip-box">
  <h4>4. 구조 및 방향성</h4>
  <p><span class="severity-mid">POSITIVE → NEGATIVE</span></p>
  <p>전반부: 꿀벌의 역할과 군집 구조 설명 (정보 제공)</p>
  <p>후반부: 꿀벌 감소의 심각성 경고 (문제 제기)</p>
  <table>
    <thead><tr><th>단계</th><th>내용</th></tr></thead>
    <tbody>
      <tr><td>도입</td><td>꿀벌은 매혹적인 곤충</td></tr>
      <tr><td>전개</td><td>여왕벌/일벌/수벌의 역할 구분</td></tr>
      <tr><td>전환</td><td>꿀벌이 사라지고 있다</td></tr>
      <tr><td>결론</td><td>꿀벌 보호의 중요성</td></tr>
    </tbody>
  </table>
</div>
```

**⑤ 선지 분석**
**모든 선지**를 정답(✅)/오답(❌)으로 구분하여 분석한다. 하나도 빠뜨리지 않는다.
```html
<!-- 정답 선지 -->
<div class="answer-box">
  <h4>✅ 정답: ③</h4>
  <p><strong>근거:</strong> "Honeybees are disappearing... this is a serious problem"</p>
  <p><strong>해석:</strong> 꿀벌이 사라지고 있으며 이것은 심각한 문제라는 내용</p>
  <p><strong>포인트:</strong> 글의 후반부에서 문제 제기 → 보호 필요성으로 연결</p>
</div>

<!-- 오답 선지 -->
<div class="warning-box" style="border-left-color: #e53e3e !important;">
  <h4>❌ 오답: ①</h4>
  <p><strong>근거:</strong> 본문에 꿀벌의 감정에 대한 언급 없음</p>
  <p><strong>해석:</strong> "꿀벌은 관심을 이끌어 산다"는 본문과 무관한 진술</p>
  <p><strong>포인트:</strong> 지문의 주제(역할 분담 + 보호)와 무관한 선지</p>
</div>
```

#### 2-B 적용 시 주의사항
- 독해 지문이 여러 개 틀렸으면, **각 지문마다 위 5단계를 반복**한다
- 지문 번호와 제목을 h3로 명시: `### 지문 27: The Life of a Honeybee`
- 선지 분석에서 정답 1개 + 오답 전부를 분석 (보통 4~5개 선지)
- 전체 해석은 지문 원문 전체를 빠짐없이 번역
- "추론 전략", "독해 풀이법", "체크리스트" 같은 메타 설명 넣지 않는다

#### 2-C. 학생 답 분석 (모든 과목 공통)

틀린 문제마다 **학생이 선택한 오답에 대한 심층 분석**을 반드시 포함한다.
해설 파트(문법/독해 모두)에서 오답 선지를 분석할 때, 단순히 "왜 틀렸는지"뿐 아니라 **"왜 학생이 이것을 골랐는지"**까지 다룬다.

포함해야 할 내용:
- **오답 선택 이유 추측**: 학생의 사고 과정을 추적하여 왜 해당 오답을 선택했는지 분석
- **사고 오류 패턴**: 어떤 유형의 오류인지 명시 (반전 함정, 개념 혼동, 부분 정보 매칭, 유사 표현 혼동 등)
- **집중 포인트**: 앞으로 유사 문제에서 주의할 점과 구체적 풀이 전략 제안

```html
<div class="tip-box" style="page-break-inside:avoid;">
  <span class="tip-label">학생 답 분석</span>
  <p><strong>왜 이 답을 골랐을까:</strong> 학생은 "renewable"과 "finite" 키워드만 보고 선형=재생, 순환=유한으로 매칭한 것으로 보입니다.</p>
  <p><strong>오류 패턴:</strong> 반전 함정 — 키워드는 맞지만 방향이 반대</p>
  <p><strong>집중 포인트:</strong> 대비/대조 지문에서 키워드를 발견하면, 반드시 "어느 쪽에 붙은 키워드인지" 확인하는 습관을 들이세요.</p>
</div>
```

### 섹션 3: 연습문제 (변형문제)
- **각 테마당 20문제** (기존 5문제에서 상향)
- **문제만 표시** — 정답/해설은 이 섹션에 넣지 않는다
- 학생이 직접 풀어볼 수 있도록 문제 사이에 여백 확보
- `line-height: 2.2`로 문제 간격 넉넉하게

#### 3-A. 문법/구문독해 연습문제
- 다양한 문제 유형 혼합: 객관식, 빈칸 채우기, 틀린 부분 찾기, 문장 완성
- 문법 규칙 적용을 테스트하는 문제 위주

#### 3-B. 독해 — 변형문제 없음
- 독해 파트는 **해설만 제공**하고 연습문제를 만들지 않는다
- 독해 해설(섹션 2-B)의 5단계 분석이 학습 자료의 전부

### 섹션 4: 정답 및 해설
- **반드시 새 페이지에서 시작** (page-break-before)
- 테마별 정답 테이블 + 간단 해설
- 테이블 형식: `# | 정답 | 해설`

## 6. 콘텐츠 작성 규칙

### 용어
- "전문 추출" → "문제 리스트" 또는 "틀린 문제 리스트"
- "발견된 항목 총정리" → "틀린 문제 리스트" 또는 "오답 분석"
- "변형문제 파트" → "연습문제" 또는 "복습문제"

### 강조 규칙 (엄격히 준수)
- 페이지당 강조 10개 이하
- 단락당 강조 2개 이하
- 문장당 강조 1개 이하
- 강조 없는 단락 60% 이상

### PDF 최적화
- 솔리드 컬러만 사용 (gradient, box-shadow, text-shadow, rgba, opacity 금지)
- `!important`로 렌더링 강제
- `page-break-inside: avoid`로 테이블/박스 분리 방지
- A4 규격: `@page { size: A4; margin: 16mm 14mm; }`

## 7. 일관성 원칙

- **같은 학생 내에서는 항상 동일한 디자인** 적용
- 학생별로 달라도 무방하지만, 한 학생의 모든 피드백지는 동일 CSS 사용
- 새 피드백지 생성 시 반드시 `03_system/templates/피드백지_template.html`의 CSS를 복사하여 사용
- 구조(섹션 순서, 헤더 형식)는 모든 학생 공통

## 8. 체크리스트

피드백지 작성 완료 후 확인:

- [ ] 표지 없음
- [ ] 목차 없음
- [ ] 첫 페이지에 리포트 헤더 + 학습 요약 카드 있음
- [ ] 요약 카드에 잘한 점 / 아쉬운 점 포함
- [ ] "전문 추출" 표현 없음
- [ ] 각 테마 변형문제 20개 이상
- [ ] 변형문제와 정답이 별도 페이지로 분리됨
- [ ] 파스텔 하늘색 팔레트 사용 (#1e3a5f, #2b6cb0, #3182ce 미사용)
- [ ] 강조 규칙 준수
- [ ] PDF 최적화 규칙 준수 (솔리드 컬러, !important 등)
